# Portfolio Table

**Sheet Name:** `Portfolio`  
**Primary Key:** `portfolio_id`  
**Row Limit (Design):** ~10 entries per student = ~1,200 rows  

---

## Purpose

Stores metadata for each student's portfolio entries. A portfolio entry represents a piece of work (code screenshot, project file, or written reflection) that a student submits to showcase their learning. The actual file is stored in **Google Drive**; this table stores the Drive link and all metadata required for display.

Portfolio entries are tied to a specific mission so teachers can assess work in curriculum context.

## Entry Type Definitions

| `entry_type` | Description | Expected File Types |
|---|---|---|
| `code` | Python code written during the mission | `.py`, `.txt`, or stored in `code_snapshot` |
| `image` | Screenshot of output, drawing, or diagram | `image/png`, `image/jpeg`, `image/gif` |
| `document` | Report, reflection, or written work | `application/pdf`, `.docx` |
| `link` | External URL (e.g. repl.it project, Canva design) | No file — `drive_url` stores the external link |
| `reflection` | Text-only learning reflection (no file) | Stored in `description` field only |

## Google Drive Integration

When a student uploads a file:

1. `PortfolioService.uploadPortfolio()` encodes the file as Base64.
2. Sends to Google Apps Script via `ApiService.upload('uploadPortfolio', file, metadata)`.
3. Apps Script `doPost` handler:
   - Decodes Base64.
   - Calls `DriveApp.getFolderById(FOLDER_ID).createFile(blob)`.
   - Sets sharing to "Anyone with link can view".
   - Saves `fileId` and `url` from Drive response.
   - Writes the `Portfolio` row with Drive metadata.
4. Returns the new `portfolio_id` and `drive_url` to the browser.

## Validation Rules

| Column | Rule |
|--------|------|
| `portfolio_id` | Must match `^PF\d{6}$`. Unique. Auto-incremented. |
| `student_id` | Must exist in `Students.student_id`. |
| `mission_id` | Must exist in `Missions.mission_id`. |
| `title` | Non-empty. Max 120 characters. |
| `description` | Max 500 characters. |
| `tags` | Max 200 characters. Each tag max 30 characters. |
| `entry_type` | Must be one of: `code`, `image`, `document`, `link`, `reflection`. |
| `drive_url` | If provided, must be a valid URL starting with `https://`. |
| `file_size_kb` | Integer ≥ 0. Must not exceed 10,240 (10 MB). |
| `code_snapshot` | Max 5,000 characters. |
| `score_ref_id` | If provided, must exist in `Scores.score_id`. |
| `teacher_grade` | Max 10 characters. |
| `teacher_feedback` | Max 500 characters. |

## API Actions

| Action Name | Method | Description |
|---|---|---|
| `uploadPortfolio` | POST | Creates a portfolio entry with optional file upload to Drive |
| `getPortfolio` | GET | Returns all entries for a student, optionally filtered by `mission_id` |
| `updatePortfolioEntry` | POST (_method: PUT) | Student updates `title`, `description`, or `is_public` |
| `deletePortfolioEntry` | POST (_method: DELETE) | Sets `is_active = FALSE` |
| `addTeacherFeedback` | POST (_method: PUT) | Teacher adds `teacher_grade` and `teacher_feedback` |
| `getClassPortfolio` | GET | Teacher views all portfolio entries for their classes |
| `getPublicGallery` | GET | Returns all `is_public = TRUE` entries for a class (shown to classmates) |
